# 📦 IMS Pro - Frontend Application

## 🏛️ System Architecture & State Management

This frontend strictly follows a component-based architecture designed for an enterprise environment:
* **Separation of Concerns:** UI components are strictly decoupled from API logic. All external communications are handled via dedicated service modules.
* **Global State Management:** React Context API is utilized for global application state (such as user authentication status and Light/Dark UI theme preferences), avoiding prop-drilling and ensuring consistent state across routes.
* **Protected Routing:** React Router DOM is configured with secure wrapper components to ensure unauthenticated users cannot access internal dashboard routes.

## 🛠️ Tech Stack & Key Technical Decisions

* **Framework:** React 18+
* **Build Tool:** Vite (Chosen over Create React App for significantly faster Hot Module Replacement (HMR), optimized builds, and better modern ecosystem support).
* **Routing:** React Router v6
* **HTTP Client:** Axios (Selected over native Fetch for its built-in request/response interceptors, simplifying automated token attachment and error handling).
* **Testing:** Vitest & React Testing Library
* **Notifications:** React Hot Toast (Provides non-intrusive, accessible toast notifications for CRUD operations and error feedback).

---

## 🔐 Authentication Strategy

Authentication state is securely managed using **JSON Web Tokens (JWT)**:
* **Token Storage:** Access tokens are securely managed in memory/local storage depending on security requirements, while Axios interceptors automatically attach them to the `Authorization` header of every outbound API request.
* **Silent Refresh:** An Axios response interceptor catches `401 Unauthorized` errors. If a token expires, the client automatically requests a new access token using the refresh token, replays the failed request, and keeps the user logged in seamlessly.

## ✨ Key Features

* **Secure Authentication Flow:** Complete login, registration, and password reset journeys with silent JWT token refreshing to keep users securely logged in without interruption.
* **Interactive Dashboard:** Real-time inventory filtering, searching by Name/SKU, and quick-toggle views for low-stock items. Features seamless CRUD modals for rapid data entry.
* **Failsafe Order Processing:** A dedicated transaction interface that validates order quantities against live server stock to prevent overselling before the request even leaves the client.
* **Rich User Profiles:** Customizable user settings including role displays, secure avatar uploads (with instant blob previews), and global Light/Dark theme preferences powered by React Context.
* **Audit Trail Visibility:** Direct access to the backend's audit engine, allowing users to view the granular history (who, what, and when) of specific inventory items directly from the dashboard.
* **Analytics & Visualizations:** Interactive charting to digest the time-series financial snapshots generated by the API.

## 🚀 Local Setup Instructions

Follow these steps to get the React application running on your local machine.

### 1. Clone the Repository

```bash
git clone https://github.com/JackScoble/ims-frontend.git
cd ims-frontend
```

### 2. Install Dependencies

Make sure you have Node.js installed, then run:

```bash
npm install
# or if you use yarn:
# yarn install
```

### 3. Configure Environment Variables

Create a `.env` file in the root of your project directory. This is required to point the frontend to your local backend API. 

```ini
VITE_API_BASE_URL=http://localhost:8000/api
```

### 4. Start the Development Server

Launch the Vite development server:

```bash
npm run dev
```

The application will now be available, typically at `http://localhost:5173/`.
